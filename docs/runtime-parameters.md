# 运行时参数设计

## 状态

- 当前状态：已完成
- 推进方式：问答式逐步定稿
- 当前问题：本轮确认点已收口

## 已确认结论

- 免费体验只提供一个很小的固定体验集
- 免费部分对所有用户一致，不做个性化
- 注册后赠送 1 套固定体验集，可反复做
- 免费体验目标不是提供完整备考闭环，而是让用户感受到产品与普通题库的差异
- 订阅后解锁完整功能
- 30 天付费期内，mock exam 次数先定为 `5`
- review pack 默认每包收纳的 ReviewTask 数量上限为 `8`（即任务数，不是题数）
- review pack 的目标题量由 `parameters.md` 的 `daily_review_size` 控制，默认 `20` 题
- 两者独立：任务数决定从任务池里取多少个复习任务，题量决定整包应完成多少题
- readiness 参数结构分为三层：
- 硬门槛参数
- 分层阈值参数
- 实验调优参数
- reminder 默认每天最多主动提醒 `1` 次
- 同类提醒需要冷却时间
- 如果用户连续 `3` 次未响应，则暂停继续提醒该类任务
- AI 成本控制默认遵循：
- 高频场景模板优先
- 题目级错因解释优先命中缓存
- topic 级说明优先命中缓存
- AI 输出默认限制为短文案
- 同用户同弱点短时间内不重复生成
- AI 失败时降级到规则文案，不影响主流程
- AI 相关配置需要支持后台可配置

## 决策记录

- 决策：免费体验只开放一个小型固定体验集，而不是完整学习闭环
- 原因：免费部分只需要证明产品价值，不应替代付费期的完整练习、复习和 mock 能力
- 决策：注册后赠送 1 套固定体验集，且可反复体验
- 原因：让用户低门槛感知产品差异，同时保持免费入口足够简单
- 决策：订阅后开放完整功能，30 天 access pass 内 mock exam 次数先定为 `5`
- 原因：mock 属于高价值验证资源，需要限额但不能太少，先用保守值启动
- 决策：review pack 默认每包 ReviewTask 数量上限为 `8`
- 原因：足以形成一轮明确复习任务，同时不会让单次复习负担过重
- 说明：`8` 是任务数上限，不是题数；目标题量（`20` 题）由 `parameters.md` 单独定义，两者独立控制
- 决策：readiness 参数结构拆成硬门槛、分层阈值和实验调优三层
- 原因：硬性准入、产品展示和后续校准应分开管理，避免参数职责混乱
- 决策：reminder 默认每天最多主动提醒一次
- 决策：若用户连续三次未响应，则暂停继续提醒该类任务
- 原因：提醒系统需要推动回流，但不能演变成打扰和骚扰
- 决策：AI 采用模板优先、缓存优先、短输出、重复生成冷却和失败降级的成本控制策略
- 原因：AI 是增强层，必须优先控制第三方成本，并避免影响主流程稳定性
- 决策：AI 参数需要支持后台配置
- 原因：后续模型、缓存、模板和成本策略都可能调整，不应依赖代码发布才能变更
