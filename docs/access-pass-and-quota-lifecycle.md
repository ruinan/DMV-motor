# Access Pass 与配额生命周期设计

## 状态

- 当前状态：已完成
- 推进方式：问答式逐步定稿
- 当前问题：本轮确认点已收口

## 已确认结论

- 固定 30 题体验集，匿名用户和注册用户均可访问，可反复做，无消耗计数（权威定义见 `docs/parameters.md`）
- 订阅后解锁完整功能
- 30 天 access pass 内 mock exam 次数先定为 `5`
- 30 天 access pass 在支付成功后立即生效
- mock 配额在“成功开始一次 mock exam”时立即扣减
- mock 进行中如果用户确认退出，本次仍视为已消耗配额
- 免费体验转付费后，账户、历史记录和当前学习状态连续保留，不做重置
- access pass 到期后：
- 不再开放完整练习、review、mock
- 仍可登录
- 仍可看到基础账户信息
- 可看到部分历史 summary / progress
- 可看到续费入口
- 固定免费体验集仍可使用
- 续费采用顺延模式
- 如果当前 access pass 还未到期，新的 30 天从当前到期时间继续往后顺延

## 决策记录

- 决策：access pass 在支付成功后立即生效
- 原因：规则最清晰，权限与账务边界最直接，避免出现“已购买但尚未开始”的复杂状态
- 决策：mock 配额在成功开始一次 mock exam 时立即扣减
- 原因：保持 quota 判断口径单一，避免用户通过反复开启后退出来规避限制，也与 mock 作为一次“验证资源消耗”的定位一致
- 决策：mock 进行中用户确认退出后，本次仍视为已消耗配额，且 attempt 直接结束
- 原因：mock 不允许像普通练习那样自由恢复；如果退出不扣配额，mock 的压力感和资源边界都会被破坏
- 决策：免费体验转付费后，不重置账户、历史记录和学习主状态，只是在原状态上立即解锁完整访问能力
- 原因：用户购买的是访问边界提升，不是重新开档；状态连续才能保证错题、复习和 readiness 判断的积累价值
- 决策：access pass 到期后保留登录、基础账户、部分历史视图和续费入口，但关闭完整学习闭环
- 原因：需要保留续费触点和状态可见性，但不能在到期后继续提供完整核心能力
- 决策：续费采用顺延模式，而不是重置
- 原因：避免用户提前续费吃亏，访问期规则也更符合直觉
