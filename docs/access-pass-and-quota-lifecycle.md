# Access Pass 与配额生命周期设计

## 状态

- 当前状态：进行中
- 推进方式：问答式逐步定稿
- 当前问题：30 天 access pass 在什么时点开始生效

## 已确认结论

- 注册用户获得 1 套固定体验集，可反复做
- 订阅后解锁完整功能
- 30 天 access pass 内 mock exam 次数先定为 `5`
- 30 天 access pass 在支付成功后立即生效
- access pass 到期后：
- 不再开放完整练习、review、mock
- 仍可登录
- 仍可看到基础账户信息
- 可看到部分历史 summary / progress
- 可看到续费入口
- 固定免费体验集仍可使用
- 续费采用顺延模式
- 如果当前 access pass 还未到期，新的 30 天从当前到期时间继续往后顺延

## 待确认问题

1. mock 配额的扣减时点
2. mock 中途退出时是否扣配额
3. 免费体验转付费后的状态切换

## 决策记录

- 决策：access pass 在支付成功后立即生效
- 原因：规则最清晰，权限与账务边界最直接，避免出现“已购买但尚未开始”的复杂状态
- 决策：access pass 到期后保留登录、基础账户、部分历史视图和续费入口，但关闭完整学习闭环
- 原因：需要保留续费触点和状态可见性，但不能在到期后继续提供完整核心能力
- 决策：续费采用顺延模式，而不是重置
- 原因：避免用户提前续费吃亏，访问期规则也更符合直觉
