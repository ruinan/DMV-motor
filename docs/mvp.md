# MVP 设计

## 1. 项目定义

### 产品一句话定义

一个帮助用户通过 California M1 permit written test 的学习工具，核心能力是围绕题目练习、错题识别和复习安排，提高笔试通过率。

### 面向对象

- 面向所有准备参加 California M1 笔试的用户

这里先不按“学过”或“没学过”细分用户。  
原因是那是后续编排策略问题，不是当前 MVP scope 问题。

### 核心价值

- 让用户持续做与考试相关的题
- 记录用户哪里做错
- 针对错误安排下一轮复习
- 用模拟考试判断是否接近可通过状态

### 内容边界

- 内容来源基于 California motorcycle handbook / runbook / 公开规则材料
- 题目、选项、解析独立编写
- 不直接照搬官方原文
- 产品定位是 based on California motorcycle handbook topics and rules
- 不是官方 DMV 产品

## 2. MVP 最大 Scope

MVP 只包含 4 个能力模块：

1. 题目练习
2. 错题记录
3. 强化复习
4. 模拟考试

AI 只作为增强，不算独立模块。  
登录只作为必要基础设施，不作为产品能力本身。

## 3. MVP 必须解决的问题

MVP 上线时，必须至少解决这 4 个问题：

1. 用户能持续做与 M1 笔试相关的题
2. 系统能记录每次答题结果
3. 系统能基于错误安排复习，而不是让用户盲刷
4. 用户能通过模拟考试判断自己是否接近正式考试水平

如果一个功能不直接帮助解决这 4 个问题，就不进入 MVP。

## 4. MVP 不做什么

- 不做社区
- 不做排行榜
- 不做视频课程
- 不做多州扩展
- 不做完整安全教育体系
- 不做复杂 agent
- 不做语音教练
- 不做骑行行为分析
- 不做内容平台

原因：这些都不属于第一版提高 M1 笔试通过率的最短路径。

## 5. MVP 功能拆解

### 功能 1：登录

作用：

- 让用户的练题记录、错题、复习进度可以保存

为什么保留：

- 没有账号体系，就没有连续学习状态，也无法做复习安排

### 功能 2：题目练习

作用：

- 提供一组基于 California M1 笔试知识范围的练习题

为什么保留：

- 这是所有后续分析和复习的入口

### 功能 3：提交答案并即时判分

作用：

- 用户答完后立刻知道对错

为什么保留：

- 不即时判分，就无法形成有效纠偏

### 功能 4：错题记录

作用：

- 记录用户做错了哪道题、错了几次、最近什么时候错

为什么保留：

- 没有错题记录，就谈不上强化复习

### 功能 5：强化复习

作用：

- 根据错题和知识点，安排下一轮更有针对性的练习

为什么保留：

- 这是产品区别于普通题库的核心能力

### 功能 6：每日复习包

作用：

- 把当天最该复习的内容收敛成一个明确任务

为什么保留：

- 让用户不需要自己决定“今天该刷什么”

### 功能 7：模拟考试

作用：

- 用接近正式笔试的方式做一次完整测试

为什么保留：

- 用户最终目标不是做题，而是通过考试

### 功能 8：模拟考试后的薄弱点总结

作用：

- 告诉用户当前最影响通过率的几个薄弱点

为什么保留：

- 模拟考试结束后，用户需要下一步行动建议，而不只是一个分数

### 功能 9：AI 简短错因解释

作用：

- 用简短语言解释用户为什么会错

为什么保留：

- 有助于把“看懂答案”变成“理解自己的错误模式”

### 功能 10：AI 推荐下一轮强化题

作用：

- 帮助给出下一轮该做什么题

为什么保留：

- 它直接服务于强化复习，不是单独炫技

## 6. MVP 页面范围

只保留最少页面：

- `/`
- `/login`
- `/dashboard`
- `/practice`
- `/review`
- `/mistakes`
- `/mock-exam`
- `/summary`

## 7. MVP 主流程

1. 用户登录
2. 用户开始练题
3. 用户提交答案，系统判分
4. 系统记录错题
5. 系统生成复习任务
6. 用户完成每日复习包
7. 用户参加模拟考试
8. 系统输出薄弱点总结和下一步复习方向

## 8. MVP 验收标准

这一部分先只定义产品验收，不定义技术验收。

### 功能验收

- 用户可以完成注册或登录
- 用户可以进入练题流程并完成作答
- 每道题提交后都能获得判分结果
- 错题可以被记录并再次查看
- 系统可以生成复习内容
- 用户可以完成一次模拟考试
- 模拟考试后可以看到分数和薄弱点总结

### 产品验收

- 用户不会把产品理解成泛安全教育平台
- 用户能明确知道下一步应该练什么
- 产品主流程可以从练题走到复习，再走到模拟考试

### 暂不定义的指标

以下指标先不在这一版文档定死：

- 刷题完成多少算“掌握”
- 模拟考试达到多少分算“ready”
- 每日复习包题量多少最合适
- AI 推荐在整体流程里占多大比重

这些要等 MVP scope 稳定后，再在后续文档中单独定义。

## 9. 关于内容导入的当前结论

当前只定一个方向，不展开实现：

- 系统应该能基于 handbook / runbook 的知识范围组织题目和复习

这意味着后续功能设计要围绕“导入规则知识后，如何支持多种复习功能”展开。  
但在当前阶段，我们先不写具体导入方式、数据结构和处理流程。
