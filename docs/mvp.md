# MVP 设计

## 1. 项目定义

### 产品一句话定义

一个面向 California motorcycle written test 的 AI 辅助刷题工具，通过识别用户错题模式和薄弱知识点，自动安排最有效的下一轮复习。

### 目标用户

- 近期要考 California M1 permit written test 的用户
- 已经刷过题，但总在相似规则、最佳行为题、场景题失分的用户
- 希望快速通过考试，而不是系统学习整套骑行教育课程的用户

### 核心价值

- 不靠海量题库，而靠精准强化
- 不只告诉用户对错，还告诉用户“为什么总在这里错”
- 不让用户重复刷已经掌握的题，把时间集中到高风险失分点

### MVP 范围

- 邮箱或 Google 登录
- 练习模式
- 即时判分与解析
- 错题本
- 今日复习包
- 模拟考试
- 学习总结
- 薄弱点分析
- AI 简短错因解释
- AI 推荐下一轮强化题

### 不做什么

- 不做社区和内容平台
- 不做视频课和长篇知识讲解
- 不做多州规则扩展
- 不做复杂智能体
- 不做骑行安全训练系统

这些内容不做的原因很直接：它们不能直接提高笔试通过率，至少在 MVP 阶段不是。

## 2. 页面列表

### `/`

作用：落地页，明确产品价值，推动注册或登录。  
为什么有帮助：让用户一进入就理解“这是帮你通过加州摩托车笔试的工具”，避免定位发散。

### `/login`

作用：Firebase Auth 登录。  
为什么有帮助：让学习记录、错题、复习队列能够跨设备保存。

### `/dashboard`

作用：学习主入口，展示今日复习包、继续练习、模拟考试入口、最近薄弱点。  
为什么有帮助：把用户下一步动作收敛到最有价值的学习任务，减少无效选择。

### `/practice`

作用：按系统推荐规则获取一组练习题，逐题作答并即时判分。  
为什么有帮助：练习是收集错误模式的入口，也是最核心的数据来源。

### `/practice/result`

作用：显示本轮练习结果、错题、错因、建议下一步。  
为什么有帮助：用户需要在每轮练习后立刻知道自己错在哪，不然很难形成纠正。

### `/review`

作用：展示今日复习包，包括到期错题、弱知识点强化题、相似规则辨析题。  
为什么有帮助：把“做过错题”变成“系统安排重复曝光直到掌握”。

### `/mistakes`

作用：错题本，按 topic、错误类型、最近错误时间筛选。  
为什么有帮助：方便用户聚焦自己最容易丢分的区域，而不是重新刷一遍所有题。

### `/mock-exam`

作用：进入一次完整模拟考试，题量、节奏、反馈方式接近真实笔试。  
为什么有帮助：真实考试通过率不只取决于知识，还取决于整体稳定性和连续答题表现。

### `/mock-exam/result`

作用：展示模拟考试得分、错题分布、薄弱点总结、建议复习计划。  
为什么有帮助：模拟考试是判断“能不能去考”的最关键页面。

### `/summary`

作用：展示近 7 天学习趋势、掌握 topic、持续薄弱 topic、推荐下一轮学习任务。  
为什么有帮助：帮助用户从零散刷题切换到明确的备考节奏。

## 3. 用户主流程

1. 用户注册并进入 dashboard
2. 系统先发一组诊断练习题，覆盖高频 topic
3. 用户提交答案，系统即时判分并记录错题
4. 规则引擎根据错误 topic、连续错误、题型错误生成 weak topic profile
5. 系统生成今日复习包
6. 用户完成复习包，系统动态更新 mastery 和 review queue
7. 用户每隔 1 到 2 天参加一次模拟考试
8. 系统基于模拟考试给出“可以继续强化哪些点，再去正式考试”

这个流程足够支撑第一版上线，因为它已经完整覆盖了：

- 获取用户水平
- 记录学习行为
- 识别错题模式
- 推送定向复习
- 用模拟考试验证通过概率

## 4. 核心功能及价值

### 练习题

价值：提供持续输入样本，让系统知道用户到底错在哪里。  
为什么对通过率有帮助：没有高质量作答数据，就无法做有效强化。

### 即时判分

价值：让用户当场纠偏。  
为什么对通过率有帮助：笔试类学习中，延迟反馈会降低纠正效率。

### 错题记录

价值：沉淀所有错误，不让问题被“下一轮刷题”掩盖。  
为什么对通过率有帮助：多数人不过不是因为完全不会，而是反复错同类题。

### 每日复习包

价值：把题目练习改造成可持续复习机制。  
为什么对通过率有帮助：通过定时复现高风险知识点，减少遗忘和重复失分。

### 模拟考试

价值：在接近真实条件下验证用户是否稳定。  
为什么对通过率有帮助：用户需要知道自己是否已经达到“可上考场”的水平。

### 薄弱点总结

价值：把零散错题归纳成可行动的问题列表。  
为什么对通过率有帮助：用户需要的是下一步学什么，而不是只是看到错题数量。

### AI 错因解释

价值：把“错了”解释成简短、可理解、可纠正的原因。  
为什么对通过率有帮助：有些用户不是不知道规则，而是不知道题目在考什么。

### AI 推荐强化题

价值：根据错因给出下一轮更有针对性的题目集合。  
为什么对通过率有帮助：减少低效重复刷题，把时间投入最可能提分的区域。

## 5. 为什么这些功能足够第一版上线

第一版不需要复杂功能，原因是通过笔试这件事的关键路径非常短：

- 用户必须做题
- 系统必须记录错误
- 系统必须识别弱点
- 系统必须安排复习
- 用户必须通过模拟考试验证

只要这 5 个环节成立，产品就已经对“提高通过率”形成闭环。社区、视频、排行榜都不是闭环必需项。
