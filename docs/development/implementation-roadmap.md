# MVP 实现推进顺序

## 1. 目标

这份文档不再继续讨论产品规则本身，  
而是定义在当前设计已经基本收口后，MVP 应该按什么顺序落地实现。

它回答这些问题：

- 先做哪一层，后做哪一层
- 哪些模块是主链路阻塞项
- 哪些能力可以延后
- 每一阶段完成后，系统至少应该达到什么状态

这份文档不定义：

- 具体代码框架选型
- 具体数据库产品
- 页面视觉设计
- 详细开发排期

## 2. 推进原则

- 优先打通“能登录、能练题、能判题、能记录错误”的最短主链路
- 优先实现规则型能力，延后 AI、提醒、导出这类增强型能力
- 优先实现共享事实状态，延后复杂快照和解释性文案
- 每一阶段都必须形成可验证的可运行切片，而不是只补零散底层

## 3. 实现阶段总览

当前建议按 6 个阶段推进：

1. 内容与基础数据层
2. 账户与访问控制
3. Practice 主链路
4. Review / Progress / Readiness 主链路
5. Mock Exam 主链路
6. AI / Reminder / Memory Export 增强层

## 4. 阶段 1：内容与基础数据层

这一阶段先把系统最基础的静态内容和核心对象落下去。

至少包括：

- `users`
- `access_passes`
- `topics`
- `questions`
- `question_variants`
- `question_related_topics`

这一阶段完成后，系统至少应该具备：

- 能管理双语题目
- 能区分主 topic 和相关 topic
- 能表达免费与付费访问状态

## 5. 阶段 2：账户与访问控制

这一阶段目标是先把“谁能进来、能用什么”做稳定。

至少包括：

- 登录态接入
- 内部 `user_id` 建立
- access pass 生效 / 到期判断
- 免费体验边界判断
- mock 配额查询能力

这一阶段完成后，系统至少应该具备：

- 用户可以登录并拿到统一账户状态
- 后端可以统一判断当前用户是否能进入完整练习、review、mock

## 6. 阶段 3：Practice 主链路

这是第一个必须跑通的完整学习闭环入口。

至少包括：

- 创建 `PracticeSession`
- 拉题
- 单题提交
- 同步判题
- 写入 `PracticeAttempt`
- 更新或创建 `MistakeRecord`

这一阶段完成后，系统至少应该具备：

- 用户可以连续练题
- 每道题提交后立刻知道对错
- 错题会沉淀为后续 review 输入

## 7. 阶段 4：Review / Progress / Readiness 主链路

这是产品区别于普通题库的核心阶段。

至少包括：

- `ReviewTask` 生成
- `ReviewPack` 收敛
- 复习流程提交
- `ProgressSnapshot` 刷新
- `ReadinessSnapshot` 刷新
- summary / next action 输出

这一阶段完成后，系统至少应该具备：

- 用户不需要自己决定今天复习什么
- 系统能明确给出下一步是继续练习、继续复习还是适合做 mock

## 8. 阶段 5：Mock Exam 主链路

这一阶段负责把“验证是否接近可考试状态”单独做出来。

至少包括：

- mock 配额校验
- 创建 `MockAttempt`
- 开始即扣减配额
- 交卷或退出结束
- 生成真实分数
- 结果回流 review / readiness

这一阶段完成后，系统至少应该具备：

- 用户可以完成一次完整 mock
- 系统可以返回分数、薄弱点和下一步复习方向

## 9. 阶段 6：增强层

这一阶段再接入非阻塞但有价值的增强能力。

包括：

- AI 短解释
- AI 下一步建议润色
- reminder task 生成与发送
- memory export 生成与加密

这一阶段原则是：

- 不改变主链路核心判断
- 失败时可降级
- 不能反向绑死前面已稳定的规则链路

## 10. 当前建议的最小首发范围

如果目标是尽快做出一个能验证产品价值的可运行版本，  
当前建议首发范围是：

- 阶段 1
- 阶段 2
- 阶段 3
- 阶段 4 的最小版
- 阶段 5 的最小版

可以延后到第二批的内容是：

- AI
- reminder
- memory export

## 11. 当前结论

当前项目已经基本完成产品规则和结构设计。  
下一步不应该继续泛化讨论，而应该按“内容基础 -> 访问控制 -> practice -> review/readiness -> mock -> 增强层”的顺序进入实现。
