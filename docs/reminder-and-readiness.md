# 提醒与 Readiness

## 1. 目标

这份文档只定义两个横切能力的产品规则：

- 复习提醒
- 学习完成度 / readiness

它们不属于单一模块，但会同时影响练习、复习、模拟考试和学习总结。

## 2. 为什么这两个能力要单独定义

这两个能力决定了产品是否只是“有人来做题”，还是“能把用户持续带到通过考试”。

原因很直接：

- 没有提醒，很多用户不会持续回来完成复习
- 没有 readiness，用户看见分数也不知道自己到底能不能去考

所以它们必须作为独立产品规则存在，而不是散落在各模块里。

## 3. 复习提醒的定位

提醒的目标不是“多发通知”，而是“在正确时间把用户拉回最值得完成的学习任务”。

提醒服务的对象主要是：

- 未完成的今日复习包
- 尚未补齐的关键薄弱点
- 适合开始下一次模拟考试的用户

提醒不应该服务这些目标：

- 纯粹增加打开次数
- 和学习状态无关的泛化召回
- 在用户明显还不适合做模拟考试时推动其考试

## 4. 提醒能力的当前产品规则

### 规则 1：后端必须具备提醒任务能力

提醒不能只依赖前端页面提示。

当前结论：

- 提醒任务的生成属于后端能力
- 前端或客户端只负责承接提醒触达

### 规则 2：提醒触达方式与端形态有关

当前结论：

- Web 可以先支持站内提醒、邮件提醒等方式
- App 后续可以扩展推送提醒
- 当前阶段先不锁死具体触达渠道

### 规则 3：提醒必须基于学习状态

提醒不能一视同仁。

当前结论：

- 用户有未完成关键复习任务时，应优先提醒复习
- 用户基础明显不稳时，不应优先提醒模拟考试
- 用户接近完成阶段时，可以提醒做模拟考试验证

## 5. 提醒服务的场景

MVP 阶段，提醒主要服务这几类场景：

### 场景 1：今日复习未完成

目标：

- 把用户拉回当天最重要的复习任务

### 场景 2：关键薄弱点未补齐

目标：

- 提醒用户不要停留在“做过很多题”，而是继续补最影响通过率的部分

### 场景 3：学习被中断

目标：

- 帮助用户回到未完成的练习或复习流程

### 场景 4：适合做下一次模拟考试

目标：

- 在合适时机引导用户进入验证阶段

## 6. 当前不在提醒里定义的内容

这份文档暂时不定义：

- 每天提醒几次
- 具体提醒时间
- 不同提醒渠道的优先级
- 多久未活跃才触发召回

这些都要等后续产品参数文档再定。

## 7. Readiness 的定位

readiness 不是考试分数本身。  
它表示的是：

- 用户当前是否已经接近正式考试可上场状态

这个概念必须和真实分数区分开。

## 8. 为什么要把分数和 readiness 拆开

原因很重要：

- 用户可能在少量题目上分数很高，但关键 topic 覆盖不足
- 用户可能练习分很高，但模拟考试稳定性不够
- 用户可能已经知道基础概念，但高混淆难点还没补齐

如果只看分数，系统很容易过度乐观。  
如果只看覆盖，又会忽略真实作答表现。

所以产品必须同时展示：

- 真实练习或模拟考试结果
- 当前 readiness / 完成度状态

## 9. Readiness 的当前产品规则

### 规则 1：不伪造真实分数

当前结论：

- 练习分数和模拟考试分数应真实展示
- 不人为压低或操控真实分数

### 规则 2：高分不自动等于 ready

当前结论：

- 用户即使某次得分很高，也不代表完全 ready
- 还要看关键 topic 是否覆盖
- 还要看关键难点是否复习完成

### 规则 3：ready 判断必须保守

当前结论：

- 在关键内容没有完成前，系统不应轻易给出完全 ready 结论
- 系统可以继续提示用户深入完成剩余高风险内容

### 规则 4：readiness 要服务下一步行动

当前结论：

- readiness 不是一个装饰性分数
- 它应该直接决定下一步建议是练习、复习还是模拟考试

## 10. Readiness 的输入来源

当前产品层只定义输入方向，不定义算法。

readiness 至少应参考：

- 练习表现
- 模拟考试表现
- 关键 topic 覆盖情况
- 关键难点是否复习完成
- 当前是否存在持续薄弱点

## 11. 学习完成度

为了让 readiness 更容易理解，产品还需要一个更偏过程型的概念：

- 学习完成度

它强调的是：

- 该学的关键内容学了多少
- 该补的关键薄弱点补了多少

### 当前产品原则

- 学习完成度更关注覆盖过程
- readiness 更关注是否接近考试状态
- 两者可以相关，但不应完全等同

## 12. 学习总结页必须表达什么

总结页至少要同时表达这几层信息：

- 你最近做得怎么样
- 你还有哪些关键内容没完成
- 你当前是否接近 ready
- 你下一步最该做什么

如果总结页只给分数，不给这几层信息，用户很难形成连续学习。

## 13. 提醒与 Readiness 的关系

提醒不能脱离 readiness 独立工作。

例如：

- 用户 readiness 很低时，提醒应偏向复习
- 用户 readiness 提升后，提醒可以偏向模拟考试
- 用户还存在明显薄弱点时，提醒应继续推动补齐而不是过早结束学习

这意味着提醒策略必须建立在 readiness 和完成度之上。

## 14. AI 在这里的角色

当前只确认 AI 可以做这些增强：

- 用更容易理解的话解释为什么当前还不够 ready
- 用简短文案说明为什么今天该优先复习某个内容

### 当前边界

- AI 不决定真实分数
- AI 不直接决定最终 readiness
- AI 只负责解释和建议增强

## 15. 当前不在这份文档里定义的内容

这份文档暂时不定义：

- readiness 的量化公式
- 学习完成度的量化公式
- 提醒触发频率
- 提醒触达渠道优先级
- readiness 分层名称

这些内容要等后续参数层文档再定义。
