# DMV Motor

一个以“帮助用户更快通过 California Motorcycle Written Test / M1 Permit Written Test”为唯一目标的 AI 辅助学习工具。

## 项目定义

### 一句话定义

基于加州摩托车手册知识点的智能刷题与强化复习产品，核心能力不是“题库数量”，而是“识别用户错题模式并安排下一轮最有效复习”。

### 目标用户

- 正在准备加州摩托车笔试的首次考生
- 已做过一些练习，但通过率不稳定的考生
- 知道基础规则，但容易在相似选项、场景题、最佳答案题上失分的用户

### 核心价值

- 用更短时间识别薄弱点，而不是盲目刷题
- 把错题从“做过”变成“被系统针对性修正”
- 让用户知道自己错在规则记忆、规则混淆、题干识别，还是被干扰项误导

## MVP 范围

### 必须做

- 登录
- 题目练习
- 提交答案并即时判分
- 错题记录
- 按知识点和错题模式生成复习
- 每日复习包
- 模拟考试
- 模拟考试后的薄弱点总结
- AI 生成简短错因解释
- AI 辅助推荐下一轮强化题目

### 明确不做

- 社区
- 排行榜
- 视频课程
- 多州扩展
- 复杂 agent
- 语音教练
- 骑行行为分析
- 泛安全教育平台

## 产品原则

- 唯一目标是提高笔试通过率
- 优先做 MVP，而不是大而全平台
- 主流程由规则引擎控制，AI 只做诊断增强和解释增强
- 所有页面、接口、数据结构都必须服务“练题 -> 识别问题 -> 定向强化 -> 提高通过率”

## 推荐技术方案

### 主流 MVP 方案

- 前端：Next.js
- API：Node.js + TypeScript
- 登录：Firebase Auth
- 数据：Firestore
- AI：Vertex AI / Gemini
- 部署：Cloud Run
- 静态资源：Cloud Storage
- 监控：Cloud Logging + Cloud Monitoring

### 为什么先选 Node.js，而不是 Spring Boot

- 对这个 MVP，核心复杂度在题目编排、用户学习状态、复习调度，不在复杂事务处理
- Firestore + Firebase Auth + Next.js 与 Node.js 的集成成本更低
- 一人或小团队能更快完成从页面到 API 到部署的闭环
- 后续如果题目管理后台、运营工具、统计计算变重，再考虑拆出独立服务

如果团队已有稳定 Java 经验，也可以把 API 层换成 Spring Boot；本文档中的领域模型和接口设计不依赖具体语言。

## 当前版本目标

第一版上线只解决一件事：

让用户在 7 到 14 天内，通过“练题 + 错题强化 + 每日复习 + 模拟考试”，显著提高通过加州摩托车笔试的概率。
